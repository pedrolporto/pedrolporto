# Implementação da nova home do GitHub

## Arquivos

- `README.md`: conteúdo principal exibido no perfil.
- `assets/profile-header.svg`: cabeçalho visual usado pelo README.

## Como publicar

1. Abra o repositório `pedrolporto/pedrolporto`.
2. Faça backup do `README.md` atual pelo histórico do Git ou copiando seu conteúdo.
3. Crie a pasta `assets` na raiz do repositório.
4. Envie `assets/profile-header.svg`.
5. Substitua o conteúdo do `README.md` pelo arquivo entregue.
6. Confirme que o repositório continua **público**.
7. Abra seu perfil para validar imagens, links, tabelas e responsividade.

## Higiene do portfólio

- Arquive ou deixe privado o repositório `teste_repositorio`, caso não tenha valor demonstrativo.
- Diferencie repositórios próprios de forks.
- Adicione `README.md`, licença, tópicos e descrição curta aos projetos que deseja destacar.
- Não publique código, dados, nomes de pacientes, regras internas, credenciais ou documentação confidencial do trabalho.
- Para cases profissionais, use diagramas e dados fictícios ou anonimizados.
- O próximo projeto público prioritário deve demonstrar Node.js, Express, API REST, testes e Docker.
